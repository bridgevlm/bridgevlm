Abstract

Visual recommendation requires retrieving items that match diverse user intents—from finding similar products to completing an outfit with complementary pieces. Recent multimodal embedding models (VLM2Vec, LamRA, MME5) learn a single embedding space optimized for similarity; they retrieve near-duplicate items effectively but collapse when the task demands cross-category complements, yielding Recall@K roughly 10× lower on complementary benchmarks. In practice, these models also define their own embedding space, preventing reuse of retrieval indices already deployed in production. The common alternative—two-stage pipelines where a large VLM (e.g., GPT-4o) verbalizes an image into a text query and a separate retrieval encoder (e.g., CLIP, SigLIP) performs nearest-neighbor search—suffers from modality misalignment: the generator and retriever are optimized independently, so generated queries drift from the optimal retrieval space.


We introduce BridgeVLM, a compact vision–language architecture that closes this gap through a differentiable retrieval bridge. The bridge maps VLM decoder hidden states directly into the embedding space of a frozen, pre-trained retrieval encoder via learnable query vectors, transformer-based refinement, and cross-model projection. Critically, the bridge is encoder-agnostic: it can target any frozen retrieval model—CLIP, Fashion-CLIP, SigLIP, or a domain-specific encoder—preserving compatibility with existing indexed catalogs without re-indexing. A multi-objective loss combining contrastive retrieval (InfoNCE) and autoregressive language modeling jointly optimizes retrieval accuracy and language fluency. We evaluate on two core tasks—Complete the Look (CTL) and Similar Item (SI) recommendation—across multiple public benchmarks. Despite being ≈200× smaller than frontier VLMs, BridgeVLM achieves state-of-the-art Recall@K on both similarity and complementary retrieval, outperforming larger VLMs, two-stage baselines, and recent multimodal embedding models, while maintaining rich language generation capability.

Overview

BridgeVLM extracts retrieval-aligned query embeddings via learnable queries refined through a cross-attention decoder and cross-modal projection. The attended queries are matched against a pre-indexed image gallery using a frozen retrieval model, returning a ranked set of recommended items. The entire pipeline is trained end-to-end, allowing retrieval supervision to flow back into the encoder.

Method

BridgeVLM is an end-to-end vision–language model that unifies language generation and image retrieval within a single architecture. Unlike two-stage pipelines, BridgeVLM introduces a differentiable retrieval bridge that aligns multimodal generation and retrieval in the retrieval model’s embedding space. The bridge is encoder-agnostic and applies to any frozen dual encoder.

Figure 3. BridgeVLM Model Architecture. The VLM backbone processes the anchor image and text prompt. The retrieval bridge extracts learnable query vectors via a transformer decoder cross-attending to VLM hidden states, then projects them through the frozen retrieval encoder’s text encoder into the shared embedding space.

Key Components

  • VLM Backbone: InternVL3-2B with a frozen vision encoder. The anchor image is tokenized into patch features and projected into the LLM token space via a two-layer MLP.
  • Retrieval Bridge: A set of N learnable query vectors refined through a 2-layer transformer decoder with cross-attention to VLM hidden states, then projected via a linear layer.
  • Retrieval Model: The projected queries are processed through the frozen retrieval encoder’s text encoder (e.g., Fashion-CLIP) to produce embeddings in the shared text–image space.
  • Training Loss: A multi-objective loss combining next-token cross-entropy (language fluency) with InfoNCE contrastive loss (retrieval alignment).

MMEB Models on CTL

Recent multimodal embedding models from the MMEB benchmark—LamRA, MME5, and VLM2Vec—predominantly retrieve items that are visually similar to the anchor rather than complementary cross-category items required by CTL. Their Recall@K on CTL is approximately 10× lower than BridgeVLM.

Pseudocode

Core forward pass of BridgeVLM’s training loop. 

# VLM – Vision Language model (InternVL3-2B)
# CLIP – Frozen retrieval model (Fashion-CLIP)
# Q_emb[M, d_llm] – M learnable query vectors
# W_proj[d_llm, d_clip] – learned projection VLM → CLIP
# Bridge_Decoder – 2-layer TransformerDecoder
# I_anchor – anchor image
# T_prompt – text prompt
# I_gt[N] – N ground truth images

# —— VLM backbone ——
input_ids, attn_mask, labels, pixels = VLM_Processor(I_anchor, T_prompt)
H_vlm = VLM(input_ids, attn_mask, pixels, labels)
H_dec = H_vlm["last_hidden_state"][-1]
M_valid = labels != -100

# —— Transformer-based refinement ——
H_att = Bridge_Decoder(tgt=Q_emb, memory=H_dec, memory_mask=M_valid)

# —— Cross-model projection ——
H_clip = H_att @ W_proj

# —— CLIP backbone ——
E_q = normalize(CLIP_text_encoder(H_clip), dim=-1)
E_i = normalize(CLIP_vision_encoder(I_gt), dim=-1)

# —— Loss ——
Sim = E_q @ E_i.T / τ
loss_ret = contrastive_loss(Sim, arange(N))
loss_ce = VLM_cross_entropy_loss(input_ids, labels)
loss = λ_ret * loss_ret + λ_ce * loss_ce
